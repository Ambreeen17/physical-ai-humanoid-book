# Urdu Technical Writing Style Guide (اردو تکنیکی تحریر کا راہنما اصول)

## Core Principles (بنیادی اصول)

1. **Clarity and Precision (واضح اور درستگی)**: Use professional, formal Urdu (Jinnah-style) appropriate for educational content. Avoid slang or overly poetic language that might obscure technical meaning.

2. **Technical Terms (تکنیکی اصطلاحات)**:
   - Keep primary technical terms in English (e.g., ROS 2, Gazebo, API, Python).
   - Provide the Urdu translation/explanation in brackets upon the first occurrence.
   - Example: "Embodied Intelligence [جسمانی ذہانت]".

3. **Code Preservation (کوڈ کا تحفظ)**:
   - **NEVER** translate code snippets, variable names, function calls, or file paths.
   - Code blocks must remain LTR (Left-to-Right).

4. **Bilinguality (دو لسانی ساخت)**:
   - Maintain a balanced side-by-side or section-by-section structure.
   - Headers should include both English and Urdu equivalents.

5. **Sentence Structure (جملے کی ساخت)**:
   - Use active voice (فعلیہ معروف) for instructions.
   - Keep sentences concise to aid readability in RTL layouts.

6. **Grammar (قواعد)**:
   - Ensure native-level Urdu syntax.
   - Correct use of honorifics and formal address in introductory text.

## Formatting Standards (فارمیٹنگ کے معیار)

- **Script**: Use standard Unicode characters (UTF-8).
- **Fonts**: Recommended fonts for rendering are "Jameel Noori Nastaleeq" for traditional look or "Noto Sans Urdu" for modern digital readability.
- **Direction**: Urdu text must be marked or handled as RTL (Right-to-Left).
